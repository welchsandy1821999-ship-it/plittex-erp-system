# 🎨 Стандарты Верстки и UI (ERP PLITTEX)

Цель документа: чистый, поддерживаемый код интерфейса без "костылей" и инлайн-мусора.

## 1. Базовые принципы
- **ZERO INLINE STYLES:** Категорический запрет на использование инлайн-стилей (`style="..."`). Любой стиль, выбивающийся из существующих классов, должен быть описан в CSS.
- **Единый источник истины:** Все глобальные стили лежат в `public/css/style.css`. Модульные стили (например, для специфичных карточек или таблиц) могут быть выделены в `public/css/modules.css`.

## 2. Flexbox Архитектура
Вместо сложных `float` или жестких `grid` (которые часто ломают адаптивность), использовать гибкий Flexbox:
- **Разметка строк:** Использовать классы `.flex-row` и `.flex-col` вместе с отступами (`gap-10`, `gap-20`, `gap-30`).
- **Выравнивание:** Использовать `.flex-wrap` для адаптивности, `.justify-between` для разнесения элементов по краям, `.flex-grow-1` для заполнения пространства.
- **Центрирование:** `.align-center`, `.justify-center`.

## 3. Цветовая палитра (CSS Variables)
Всегда использовать встроенные переменные вместо хардкода hex-кодов. Основные:
- Фоновые блоки: `var(--surface)`, `var(--surface-alt)`
- Текст: `var(--text-main)`, `var(--text-muted)`
- Акценты: `var(--primary)` (синий), `var(--success)` (зеленый), `var(--danger)` (красный).
- Бордеры: `var(--border)`

## 4. Дизайн-контролы и Размеры
- **Отмена подчеркивания ссылок (Anti-Underline):** Категорически запрещено добавлять подчеркивание к ссылкам (через style или тег `<u>`). Ссылки в ERP не подчеркиваются. Применять `text-decoration: none` или глобальный класс.
- **Размеры элементов:** Высота инпутов и базовых кнопок должна быть приведена к стандарту **38px** или **40px** (по умолчанию применяется через класс `.input-modern`).
- **Кнопки:** Для кнопок использовать общие классы: `.btn`, `.btn-blue`, `.btn-outline`. 
- **Шрифты интерфейса:** базовые размеры `font-13`, `font-14` (для таблиц и кнопок), `font-16/18` для итоговых сумм.

## 5. Скрытие элементов
Если элемент скрыт по умолчанию (например, калькулятор при загрузке), использовать базовый CSS-класс `.sales-hidden` или аналогичный (`display: none !important`) в HTML, и управлять его снятием через JS (`classList.remove()`). Сам элемент в CSS должен иметь свой естественный `display` без необходимости переопределять его блочно в JS.
