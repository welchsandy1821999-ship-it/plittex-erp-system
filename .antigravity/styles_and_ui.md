# 🎨 Стандарты Верстки и UI (ERP PLITTEX)

Этот документ регламентирует подход ИИ-ассистента к любым задачам по визуалу, верстке и стилям. Главная цель: чистый, поддерживаемый код интерфейса без "костылей".

## 1. Базовые принципы
- **ZERO INLINE STYLES:** Категорический запрет на использование инлайн-стилей (`style="..."`). Любой стиль, выбивающийся из существующих классов, должен быть описан в CSS.
- **Единый источник истины:** Все глобальные стили лежат в `public/css/style.css`. Модульные стили (например, для специфичных карточек продаж или производства) лежат в `public/css/modules.css`.

## 2. Flexbox Архитектура
Вместо сложных float или жестких grid (если они ломают верстку, как было с `sales-checkout-grid`), использовать гибкий Flexbox:
- **Разметка строк:** Использовать `.flex-row` с отступами (`gap-10`, `gap-20`, `gap-30`).
- **Сжатие и растяжение:** Использовать `.flex-wrap` для адаптивности, `justify-between` для разнесения кнопок по краям, `flex-grow-1` для заполнения пространства.

## 3. Цветовая палитра (CSS Variables)
Всегда использовать встроенные переменные вместо хардкода hex-кодов:
- Фоновые блоки: `var(--surface)`, `var(--surface-alt)`
- Текст: `var(--text-main)`, `var(--text-muted)`
- Акценты: `var(--primary)` (основной синий), `var(--success)` (зеленый), `var(--danger)` (красный/удаление).
- Бордеры: `var(--border)`

## 4. Размеры элементов (Input/Buttons)
- **Точность размеров:** Высота инпутов и базовых кнопок в модулях оформления (sales/production) должна быть приведена к стандарту **38px** или **40px**. Никогда не раздувать элементы без прямой команды пользователя.
- **Шрифты:** Стандартные шрифты интерфейса `font-13`, `font-14` (для кнопок), `font-16/18` для итоговых сумм.
- Для кнопок использовать общие классы: `.btn`, `.btn-blue`, `.btn-outline` + кастомный класс для точечных модификаторов (например, `.sales-btn-add`).

## 5. Скрытие элементов
- Если элемент скрыт по умолчанию (например, калькулятор), использовать базовый класс `.sales-hidden` (`display: none !important`) в HTML, и управлять его снятием через JS. Сам элемент в CSS должен иметь естественный `display` (например, `display: flex`).
