# Архитектурный манифест проекта (ERP PLITTEX)

Этот файл задает только архитектурные принципы проекта и не дублирует операционные правила агента.

## 1) Источники истины
- Процесс работы агента и ограничения исполнения: `.cursorrules`.
- Архитектура и устройство системы: `erp_architecture_tree.md`.
- Функциональные и технические детали модулей: `erp_technical_docs.md`.
- Специализированные стандарты:
  - База данных: `.antigravity/db_protocol.md`
  - UI и верстка: `.antigravity/styles_and_ui.md`

## 2) Базовые архитектурные требования
- Работать от фактического кода, не от предположений.
- Сверять изменения по цепочке `DB -> routes -> frontend (EJS/JS)`.
- Не хардкодить системные идентификаторы (склады, счета, служебные сущности).
- Любые найденные расхождения между кодом и docs исправлять в документации в рамках той же задачи.

## 3) SSR/EJS и клиентская логика
- Помнить, что EJS исполняется на сервере до отправки HTML.
- Не смешивать серверную и клиентскую логику в одном блоке без явной границы ответственности.
- Для пользовательских строк использовать безопасный вывод и экранирование по контексту.

## 4) Статические ресурсы
- Статика хранится в `public/`.
- Модульные скрипты/стили должны быть организованы по существующей структуре проекта.
- Внешние CDN добавлять только при явной необходимости и после согласования.
