# 📜 Главные Заповеди ИИ-Ассистента (ERP PLITTEX)

Эти правила имеют наивысший приоритет. Любые действия ИИ должны сверяться с этим списком.

**1. Интроспекция и Первоисточник Истины (Self-Healing Docs)**
Никаких догадок. Схему БД читай через встроенный SQL-инструмент (без терминальных скриптов!). Если база недоступна — смотри папку `migrations/`. Файлы `erp_*.md` используй как общую карту, но если найдешь расхождение между картой и реальным кодом/БД, твоя обязанность — автоматически обновить Markdown-файл документации, чтобы она не устаревала.

**2. Двухфазный контракт (Autonomous Edition)**
Анализ -> Утверждение -> Автономное исполнение.
Сначала обсуждаем проблему и план действий. После того как пользователь пишет "Ок/Делай", ИИ обязан выполнить ВЕСЬ объем работы (замена кода в 1, 3 или 5 файлах) автономно и без остановок. Дробить работу на микро-вопросы в процессе запрещено.

**3. Zero-Terminal Policy и Метод "Stealth API"**
Строго запрещено использовать терминал (`bash`, `PowerShell`, `node -e`) для статического поиска, файловых операций или скриптов (он локально зависает). Использовать встроенные утилиты `grep_search`/`view_file`. Если требуется запустить скрипт, удалить файлы или выполнить фоновую задачу — использовать метод "Stealth API": временно внедрить маршрут в `web.js` -> триггернуть его через внутренний HTTP-запрос -> удалить маршрут. Terminal остается только для `npm run dev`.

**4. Связка данных (Тройной контроль)**
Всегда проверяй соответствие названий переменных и логики на всех трех уровнях системы: **База данных (SQL/ migrations)** -> **Бэкенд (роуты)** -> **Фронтенд (EJS/JS запросы)**. Например, если в базе склад называется `type='reserve'`, на фронтенде нельзя жестко проверять `id === 7`.

**5. Стандарт UI (Flex и Zero Inline Styles)**
Категорический запрет на инлайн-стили (`style="..."`). Вся вёрстка строится на семантических классах и Flexbox (`flex-row`, `gap-20`). Стили выносятся строго в `public/css/modules.css` (или `style.css`).

**6. Финансовая и Складская Точность (Big.js)**
Любые вычисления денег или складских остатков на фронтенде и в бэкенд-маршрутах ВСЕГДА должны оборачиваться в библиотеку `Big.js` (или `Math.round/parseFloat` с жесткой защитой). Игнорирование этого приведет к ошибкам плавающей точки (`0.1 + 0.2`).

**7. Auto-Run Protocol (Commander's Trust)**
Используй `SafeToAutoRun: true` для всех команд терминала, которые не являются деструктивными, чтобы не отвлекать пользователя кликами "Run". (Сборки, рестарты серверов, линтеры).

**8. Сквозной фокус**
Мы занимаемся ровно одной задачей до тех пор, пока пользователь прямо не скажет: "Мы закончили, идем дальше". До этого момента 100% внимания ИИ сосредоточено на доведении текущей задачи до идеала. Не забегать вперед, не предлагать переключения на другие модули.
