## 2. Блок "Костылей" (`modules.css` строки 5000-5025)

Это самая опасная часть, из-за которой сломалась кнопка `🔄 Обновить`. Здесь классы переопределены через `!important`, создавая скрытые баги по всей системе.

### 💣 Троянские классы (Причина багов)
1. **`.p-5 { padding: 4px 6px !important; }`**
   * **Проблема:** В `theme.css` класс `.p-5` означает `padding: 5px;`. Кто-то перебил базовый отступ, чтобы сделать инпуты в таблице `inventory.js` "поменьше".
   * **Решение:** Удаляем этот `!important`. Создаем легальный класс `.input-sm { padding: 4px 6px; font-size: 13px; }` в `components.css`.
2. **`.h-32 { height: 32px !important; }`**
   * **Проблема:** Именно он ломает логику кнопки `.btn`. Стили кнопки требуют ~34px высоты, а `.h-32` жестко обрезает её через `!important`.
   * **Решение:** Удаляем. Для кнопок создаем `.btn-sm { padding: 4px 12px; height: 32px; }`.

### 🧩 Одноразовые Flex-утилиты
Классы `.flex-12`, `.flex-15`, `.flex-08` и т.д.
* **Анализ:** Мой поиск показал, что они используются **исключительно** в файле `public/js/inventory.js` (строка 1134+). 
* **Решение:** Их безопасно удалить из глобального CSS. В `inventory.js` мы просто применим ту "Умную систему ширин", которую сделали для Карточки партии вчера.

### 🟢 Безопасные утилиты таблиц
* `.table-fixed`, `.bg-danger-light`, `.border-top-dotted` (используются в таблице `inventory.ejs`).
* **Решение:** Переносим в `theme.css`.

## 🏗️ План безопасного рефакторинга (Action Plan)

Чтобы ни одна страница не отвалилась, я буду действовать так:

1. **Создание базы:** Пишу классы `.btn-sm` и `.input-sm` в `components.css`. Переношу нужные классы (цвета, ширину) из "мусора" в чистый `theme.css`.
2. **Зачистка CSS:** Полностью сношу блоки AUDIT-011 из `components.css` и блок 5000-5025 из `modules.css`. На этом этапе костыли физически исчезают, уступая место чистой системе.
3. **Обновление HTML:** 
   В файлах `inventory.ejs` и `production.ejs` заменяю костыльные сборки `<button class="btn h-32 px-15">` на `<button class="btn btn-sm">`. 
   Меняю `<input class="input-modern p-5 h-32">` на `<input class="input-modern input-sm">`.
4. **Обновление JS-рендеров:** 
   В `public/js/inventory.js` заменяю инлайновые `.flex-12` на стандартные CSS селекторы `.filter-period { flex: 1.2 }`.

### 🛡️ Гарантия безопасности
Ни один CSS-класс, который используется в JS или HTML, не будет просто так "стерт в никуда". Либо он останется без `!important` (так как он дубликат оригинала), либо будет переименован в `sm` модификатор, а я программно изменю его во всех файлах шаблонов по всему проекту.
