// turbo-all

1. **Сбор Контекста (База знаний):** Чтение правил (`.antigravity/rules.md`), протоколов и словаря. Не угадывать архитектуру.
2. **Анализ Кода (Только статика):** Использовать `grep_search` и `view_file` для поиска связей. Запрет на терминальные скрипты для интроспекции.
3. **Планирование (ФАЗА УТВЕРЖДЕНИЯ):** Создать детальнейший план-артефакт `implementation_plan.md`. Строго остановиться и дождаться утверждения пользователя.
4. **Автономное Выполнение (ФАЗА ИСПОЛНЕНИЯ):** После получения "ок", выполнить весь объем работ **САМОСТОЯТЕЛЬНО и без остановок**.
   - Редактирование файлов (`replace_file_content`) производится непрерывно.
   - Любые тестовые команды в терминале строго сопровождаются флагом `SafeToAutoRun: true`, чтобы не отвлекать пользователя кликами на кнопку "Run". 
   - Остановка допускается только при непредвиденном сбое.
5. **Верификация и Сдача:** Остановиться и запросить ручную визуальную проверку у пользователя. Следовать правилу "Сквозной фокус".
